## 2. 详细功能规格

### 2.1 CLI 基础命令体系

| 命令 | 说明 |
|---|---|
| `jaicode plan "<需求>"` | 进入 Architect 模式 |
| `jaicode code "<任务>"` | 进入 Code 模式 |
| `jaicode debug "<命令>"` | 进入 Debug 模式 |
| `jaicode ask "<问题>"` | 进入 Ask 模式 |
| `jaicode rollback` | 恢复最近一次文件备份 |
| `jaicode config --lang zh\|en` | 切换语言 |
| `jaicode --help` / `-h` | 查看帮助 |
| `jaicode --version` / `-v` | 查看版本号 |

**通用参数（适用于 plan/code/debug/ask）：**
| 参数 | 说明 | 默认值 |
|---|---|---|
| `--yes` / `-y` | 跳过Diff确认，直接应用（用于CI场景） | 关闭 |
| `--verbose` | 输出详细调试日志 | 关闭 |
| `--quiet` | 抑制过程信息，仅保留关键结果 | 关闭 |
| `--max-retries <N>` | 仅Debug模式，最大自动修复重试次数 | 5 |
| `--lang zh\|en` | 单次命令临时指定语言，不改变全局配置 | 跟随全局配置 |

**验收标准：**
- 未知命令/参数需给出双语友好提示及最相似命令建议（如 `did you mean 'code'?`）
- `--help` 输出内容必须双语对齐，不因语言切换而缺失子命令说明

---

### 2.2 Architect 模式（`plan`）详细规格

| 项目 | 规格 |
|---|---|
| 输入 | 自然语言需求描述（必填），可选项目路径参数 |
| 权限 | 只读文件目录 + 允许联网搜索技术方案（复用LLM工具调用能力） |
| 处理流程 | 读取项目结构 → 构建上下文 → 调用LLM生成设计方案 → 格式化输出 |
| 输出 | 生成 `ADR.md`（Architecture Decision Record）文件，中英双语内容随当前语言设置生成 |
| 边界条件 | 项目为空目录时，仍可基于纯需求描述生成方案，需在输出中注明"未检测到现有代码结构" |
| 异常处理 | 联网搜索失败时降级为仅基于本地上下文生成方案，并提示用户网络异常 |

**验收标准：**
- 生成的 `ADR.md` 包含：背景、方案对比、推荐方案、风险点四个必备章节
- 不覆盖已存在的 `ADR.md`，而是追加版本号（如 `ADR_v2.md`）或提示用户确认覆盖

---

### 2.3 Code 模式（`code`）详细规格

| 项目 | 规格 |
|---|---|
| 输入 | 自然语言任务描述 |
| 权限 | 读写文件系统 |
| 处理流程 | 读取上下文 → LLM生成变更方案 → 生成Diff → 用户确认 → 写入（自动备份）→ 反馈结果 |
| 输出 | 修改后的文件 + 终端Diff展示 |
| 单次任务变更范围限制 | 单次任务默认最多修改 **20个文件**，超出需二次确认（防止大范围误改） |
| 异常处理 | LLM生成的Diff无法应用于当前文件版本（如文件已被外部修改）时，提示"文件已变化，请重新执行任务"并终止 |

**验收标准：**
- 拒绝确认（选择N）后，文件系统不产生任何变化，任务标记为"已取消"
- Diff展示中必须包含文件路径、变更行号、上下文行（前后各2行），不可仅展示变更行本身

---

### 2.4 Debug 模式（`debug`）详细规格

| 项目 | 规格 |
|---|---|
| 输入 | 待执行的终端命令（如测试/编译命令） |
| 权限 | 读写文件系统 + 执行终端命令（限制在项目目录内） |
| 处理流程 | 执行命令 → 捕获stdout/stderr → 失败则生成修复方案 → Diff确认 → 应用 → 重新执行 → 循环直至成功或达到重试上限 |
| 重试上限 | 默认5次，可通过 `--max-retries` 调整，达到上限后终止并给出人工介入建议 |
| 安全限制 | 执行的命令仅限项目目录内，禁止执行含 `rm -rf /`、`sudo` 等高危模式的命令，检测到则拒绝执行并提示 |

**验收标准：**
- 每次重试必须展示"第N/MAX次尝试"进度，不可静默重试
- 达到重试上限时必须保留最后一次的错误日志路径，便于用户人工排查
- 高危命令检测覆盖：删除根目录、格式化磁盘、权限提升类命令（具体黑名单规则见附录）

---

### 2.5 Ask 模式（`ask`）详细规格

| 项目 | 规格 |
|---|---|
| 输入 | 自然语言问题 |
| 权限 | 无文件写权限；可选只读访问当前目录用于回答上下文相关问题 |
| 处理流程 | 直接调用LLM对话，无Diff/无文件操作 |
| 输出 | 终端文本回答 |

**验收标准：**
- Ask模式下任何情况都不得触发文件写入，需在单元测试中显式覆盖"防越权"用例

---

### 2.6 文件安全与备份回滚

| 功能点 | 规格 |
|---|---|
| 自动备份 | Code/Debug模式写入前，将原文件复制到 `.jaicode_backup/<timestamp>/` |
| 回滚指令 | `jaicode rollback` 恢复最近一次备份；`jaicode rollback --list` 查看备份历史 |
| 备份保留策略 | 本地保留最近 **20次** 备份记录，超出自动清理最旧记录 |
| 冲突处理 | 回滚时若目标文件已被外部修改，提示差异并要求二次确认 |

---

### 2.7 I18n 引擎

| 功能点 | 规格 |
|---|---|
| 自动检测 | 启动时读取系统语言环境变量，匹配zh/en，其他语言默认降级为英文 |
| 手动切换 | `jaicode config --lang zh\|en`，写入本地配置持久化，重启后生效 |
| AI Prompt对齐 | System Prompt模板按当前语言动态选择版本，覆盖代码注释、错误解释等AI输出内容 |
| 资源缺失处理 | 若某文案在目标语言资源中缺失，兜底显示英文并记录日志（便于后续补全翻译） |

---

### 2.8 LLM Provider Adapter

| 功能点 | 规格 |
|---|---|
| 抽象接口 | `ILLMProvider.chat(messages, tools): Promise<Response>` |
| 本期Provider | 至少接入1个LLM供应商完成闭环验证 |
| BYOK | 用户可通过 `jaicode config --api-key <key>` 设置自有密钥，本地加密存储 |
| 超时与重试 | 单次LLM请求超时时间默认60秒，超时后重试1次，仍失败则终止任务并提示 |
| 流式响应 | Code/Debug模式的LLM响应建议采用流式展示，降低用户等待感知 |

---

### 2.9 本地日志

| 功能点 | 规格 |
|---|---|
| 存储位置 | `~/.jaicode/logs/<date>.jsonl` |
| 记录内容 | 命令、时间戳、执行结果、错误堆栈（不记录用户代码内容全文，仅记录文件路径与diff摘要） |
| 日志级别 | `--verbose`输出DEBUG级别到终端；默认仅INFO级别落盘 |

---

### 2.10 授权占位功能（`NoOpLicenseProvider`）

| 功能点 | 规格 |
|---|---|
| 行为 | 所有 `validate()` 调用始终返回 `{isValid: true, plan: 'trial'}`，无网络请求 |
| 启动提示 | CLI启动时展示"当前为内测版本，暂不限制使用"提示（可通过 `--quiet` 抑制） |
| 接口保留 | 所有Orchestrator对授权状态的判断必须走 `ILicenseProvider` 接口，不允许硬编码跳过，确保二期切换零改动 |

---

### 2.11 意向收集提示功能

| 功能点 | 规格 |
|---|---|
| 触发条件 | 每成功完成 **10次** Code/Debug任务后触发一次（本地计数，非云端统计） |
| 展示形式 | 任务完成后追加一段简短提示 + Waitlist链接，双语 |
| 关闭方式 | 支持 `jaicode config --disable-tips` 永久关闭该提示 |

## 3. 异常场景与容错设计

| 异常场景 | 处理方式 |
|---|---|
| 网络中断（LLM请求失败） | 提示"网络连接失败，请检查网络后重试"，不产生文件变更 |
| LLM返回格式异常/无法解析 | 记录原始响应到日志，提示用户"AI响应解析失败，请重试"，任务标记失败 |
| 目标文件被外部程序占用/权限不足 | 捕获文件系统异常，提示具体文件路径与权限问题，不中断整个CLI进程 |
| 项目目录内文件数量过大（如>10万文件） | ContextBuilder 启用采样/索引策略，避免全量扫描导致卡顿，超过阈值时提示用户可指定子目录 |
| Debug模式命令本身缺失（如未安装npm） | 捕获"command not found"类错误，直接终止并提示环境问题，不计入重试次数 |
| 用户中途 `Ctrl+C` 中断 | 已应用的变更保留备份记录，未应用的变更不落盘，退出时提示当前任务状态 |

## 5. 验收标准 Checklist

- [ ] 四种Agent模式均可独立运行，且权限边界符合规格（Ask无写权限验证覆盖）
- [ ] Debug模式重试上限、高危命令拦截均有自动化测试用例覆盖
- [ ] 所有写文件操作均有对应备份，`rollback` 可正确恢复
- [ ] 中英文切换后，CLI文案与AI输出语言均对齐，无缺失字段
- [ ] 断网情况下核心Agent功能之外的部分（如意向提示）不阻塞主流程
- [ ] `NoOpLicenseProvider` 通过单元测试验证接口契约，为二期替换做好准备
