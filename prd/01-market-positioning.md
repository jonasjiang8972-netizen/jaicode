## 1. 市场分析

### 1.1 全球市场格局：三大架构流派

2026年，AI编程工具市场已从"代码补全"演化为"自主智能体"竞争，行业普遍按产品形态分为三个流派：

| 流派 | 代表产品 | 特点 | 迁移成本 |
|---|---|---|---|
| **IDE 锁定型** | Cursor、Windsurf(现Devin Desktop)、Kiro、Antigravity | 深度编辑器整合，体验打磨精细，但配置/规则文件不可迁移 | 高 |
| **编辑器扩展型** | GitHub Copilot、Cline、Zed | 依附现有编辑器生态，开源或跨IDE覆盖广 | 中 |
| **终端便携型（CLI-first）** | Claude Code、Codex CLI、Grok Build | 不依赖特定编辑器，可在任意终端/CI流水线中运行，脚本化程度高 | 低 |

其中**终端便携型是2026年上半年增速最快、机构评测认可度最高的细分品类**，被普遍认为是"自动化重度场景"（多仓库协作、CI/CD集成、无人值守长任务）的最佳选择。这正是本产品选择"本地CLI"形态而非GUI客户端的市场依据。

### 1.2 全球主流产品定价参考

| 产品 | 形态 | 个人版定价 | 团队版定价 | 备注 |
|---|---|---|---|---|
| Cursor | IDE锁定 | $20/月 | $40/座席 | 估值约$50B，ARR超$2B |
| GitHub Copilot | 扩展型 | $10/月 | $19-39/座席 | 装机量最大（约1500万开发者），转向按量AI Credit计费 |
| Claude Code | 终端便携 | 含于Pro $17-20/月 | Team $20/座席（年付） | Max版$100-200/月，重度自动化场景性价比突出 |
| Windsurf/Devin Desktop | IDE锁定 | 免费/Pro $20/月 | Teams $80基础+$40/座席 | 被Cognition收购后更名 |
| Codex CLI / Cline | 终端便携/BYOK | 免费或按API量计费 | - | 强调vendor independence，成本自控 |

**趋势判断**：定价普遍从"固定订阅"转向"订阅+用量"混合计费，纯粹的**买断制License Key**模式在国际主流工具中反而少见——多数厂商选择SaaS订阅以获取持续收入与用户行为数据。这意味着"授权码买断/订阅可选"的商业模式，对于**预算敏感型个人开发者与不愿被云端锁定的企业客户**仍是有效差异化点，但不是当前主流打法，需要谨慎验证付费意愿。

### 1.3 中国市场格局

| 维度 | 数据 |
|---|---|
| 市场规模（2025） | 约24.5亿元人民币，同比增长187.3% |
| 市场集中度 | CR3（前三名）占比约72%，高度集中 |
| 企业付费主力 | 企业开发团队占比约45.3% |
| 渗透率 | 国内AI编程工具渗透率超78% |
| 头部玩家 | 字节跳动 Trae、阿里云 通义灵码、百度 文心快码、腾讯云 CodeBuddy、智谱 CodeGeeX |

**国产工具竞争格局特点：**
- 几乎全部采用**免费或极低价策略**换取用户规模（Trae、CodeGeeX完全免费，通义灵码基础版免费）
- 产品形态以**IDE插件或独立IDE**为主（Trae、CodeBuddy IDE、Lingma IDE），**真正对标Claude Code的轻量终端CLI品类在国内基本处于空白状态**
- 中文语境理解显著领先国际工具（如中文注释生成准确率、安全基准测试成绩），但在Agent自主能力、多文件协同、复杂推理上普遍承认落后于Cursor/Claude Code
- 商业化路径高度依赖云生态绑定（阿里云账号、腾讯云账号），**缺乏面向"涉密/断网/私有化"企业客户的成熟离线授权方案**

### 1.4 市场空白与机会点

综合国际与国内格局，识别出以下机会窗口：

| 机会点 | 说明 | 竞争强度 |
|---|---|---|
| **本地优先 + 终端CLI + 中英双语** | 国内该品类几乎空白，国际该品类被Claude Code/Codex占据但普遍无中文本地化深度优化 | 低-中 |
| **离线/私有化授权能力** | 国内头部工具普遍绑定云账号，缺乏断网环境企业级方案；国际工具的企业私有化多为高价定制合约 | 低 |
| **买断制License Key + 订阅可选** | 主流玩家几乎全部押注纯订阅制，对"不愿被云锁定"的客户群体存在服务空白 | 低 |
| **出海/跨境团队场景** | 中英团队混合协作需求增长，但现有中文工具国际化程度低，国际工具中文体验弱 | 中 |

**结论**：本产品不应定位为"再造一个Cursor/Trae"，而应聚焦 **"本地部署 + 终端原生 + 双语对等 + 授权可离线"** 这一交叉细分市场，主要面向：出海技术团队、涉密/合规敏感型企业、以及厌倦SaaS订阅锁定的独立开发者。

## 2. 产品定位（重新定义）

### 2.1 一句话定位
面向**隐私敏感型企业与中英双语开发团队**的本地优先终端 AI 编程 Agent——代码不出本地设备，授权可离线激活，中英文体验完全对等。

### 2.2 差异化价值主张（对比矩阵）

| 维度 | Claude Code / Codex CLI | 国内IDE型工具（Trae/灵码） | **本产品定位** |
|---|---|---|---|
| 部署形态 | 终端CLI ✓ | IDE插件/独立IDE | 终端CLI ✓ |
| 中文本地化 | 弱 | 强 ✓ | 强 ✓ |
| 离线/私有化授权 | 企业合约定制 | 基本无 | **标准化离线激活能力** ✓ |
| 授权模式 | 纯订阅 | 免费/云绑定订阅 | **买断+订阅可选** |
| 数据合规 | 云端为主 | 云端为主 | **本地优先** ✓ |

### 2.3 目标用户与场景（更新）

| 用户画像 | 核心诉求 | 对应机会点 |
|---|---|---|
| 出海技术团队 | 中英文团队协作，工具体验一致 | 双语对等 |
| 金融/政务/军工企业 | 代码不出域，可断网使用 | 离线授权+本地部署 |
| 独立开发者/自由职业者 | 不想被订阅锁死，一次买断长期用 | 买断制License Key |
| 中小团队负责人 | 明确的团队授权与用量管控 | 团队授权后台（后续阶段） |

## 3. MVP 功能范围界定（本期实现 vs 预留）

### 3.1 本期必须实现（Phase 1 核心）
聚焦**Agent核心能力闭环**与**底层架构可靠性**，验证产品是否真正好用，再谈商业化：

- [x] Architect / Code / Debug / Ask 四种 Agent 模式的完整任务循环
- [x] 跨平台（Mac/Windows）独立二进制分发
- [x] 中英文双语 i18n 引擎（终端交互、AI回复语言对齐）
- [x] 文件系统安全沙箱（Diff确认、自动备份回滚）
- [x] LLM Provider Adapter（至少支持1-2个模型供应商，含 BYOK）
- [x] 本地结构化日志

### 3.2 本期预留、不深入实现（Reserved Module）

授权激活与官网支付模块**在架构层面预留标准接口，但本期不投入完整实现**，具体处理方式：

| 模块 | 本期处理方式 | 二期计划 |
|---|---|---|
| 授权激活（License） | 定义 `ILicenseProvider` 接口，默认提供 `NoOpLicenseProvider`（始终返回"已激活"），不阻塞开发/内测使用 | 接入真实云端鉴权服务 + 离线证书机制 |
| 官网支付 | 暂不开发支付页面，仅在Web控制台预留"购买"入口占位（跳转至人工联系方式或Waitlist） | 接入 Stripe / 国内发卡平台，打通购买-激活链路 |
| 团队授权管理后台 | 不开发 | 视团队版需求排期 |

> **理由**：先验证核心Agent能力的产品市场契合度（PMF），避免过早投入商业化基础设施；但架构必须为后续快速接入授权与支付预留清晰的扩展点，避免二期推倒重来。

## 6. 风险与应对（更新）

| 风险 | 说明 | 应对 |
|---|---|---|
| Agent能力落后头部产品 | Claude Code/Cursor在复杂推理、多文件协同上持续领先 | MVP阶段聚焦"够用且可靠"而非"全面对标"，优先打磨Debug自动修复循环体验 |
| 无商业化模块导致早期无法验证付费意愿 | 预留模块策略可能延迟商业验证 | 通过Waitlist/人工联系收集付费意向，作为二期启动商业模块的信号 |
| 国产云厂商生态绑定优势 | 通义灵码/CodeBuddy背靠云账号体系，获客成本低 | 差异化定位于"不愿被云绑定"客户，而非正面竞争云生态用户 |
| 终端CLI品类用户教育成本 | 国内开发者更习惯IDE插件形态 | 前期文档/教程需重点讲解CLI相较IDE插件的自动化优势 |
