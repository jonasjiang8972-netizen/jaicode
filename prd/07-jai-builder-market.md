## 1. 市场分析

### 1.1 低代码/无代码市场格局

| 层级 | 代表产品 | 定位 | 用户群 |
|---|---|---|---|
| **企业级** | OutSystems、Mendix、微软 Power Platform | 全栈应用开发 | 企业 IT 团队 |
| **应用级** | Bubble、Webflow、Framer | 网站/应用搭建 | 中小团队、独立开发者 |
| **组件级** | Retool、Appsmith、ToolJet | 内部工具/仪表盘 | 开发者 |
| **页面级** | 阿里宜搭、腾讯微搭、百度速搭 | 表单/审批/页面 | 业务人员 |
| **设计级** | Figma、即时设计、MasterGo | UI 设计 + 代码生成 | 设计师 |

### 1.2 市场空白分析

当前市场存在以下空白点：

| 空白点 | 说明 | 竞争强度 |
|---|---|---|
| **AI + 低代码融合** | 现有低代码工具依赖手动拖拽，AI 驱动生成能力弱 | 中 |
| **中文语境优先** | 国内工具 UI 友好但扩展差，国际工具中文支持弱 | 低 |
| **终端原生 CLI 驱动** | 无 CLI + TUI 形态的低代码工具（均为 Web GUI） | 极低 |
| **开发者优先的轻量工具** | Retool 太复杂，Webflow 太重，缺少轻量级选项 | 中 |
| **插件/扩展生态** | 开源低代码工具缺少 skill/marketplace 机制 | 低 |

## 2. 产品定位

### 2.1 一句话定位
面向开发者的 AI 驱动终端低代码页面设计工具 — 可视化拖拽 + 自然语言生成 + 完整代码导出。

### 2.2 核心差异化

1. **终端原生 TUI 界面** — 在终端中完成页面设计，无需浏览器
2. **AI 自然语言驱动** — 输入描述自动生成页面结构和组件
3. **完整代码导出** — 生成可直接运行的 React/Vue/HTML 代码
4. **Jaicode 生态集成** — 复用 Jaicode 的 Provider 配置、Marketplace、Skills
5. **项目级配置** — 支持模板、主题、组件库扩展

## 3. MVP 功能范围

### 3.1 本期实现（Phase 1）

- [ ] 终端 TUI 画布渲染（组件树 + 属性面板）
- [ ] 拖拽式组件添加（键盘驱动）
- [ ] 预设组件库（文本、按钮、输入框、图片、卡片、列表、表单）
- [ ] 组件层级树可视化
- [ ] 属性编辑器（文本内容、样式、尺寸、颜色）
- [ ] 实时预览（字符画 + 描述）
- [ ] AI 自然语言生成页面（基于 Jaicode Provider）
- [ ] 代码导出（React / Vue / 纯 HTML）
- [ ] 项目配置保存/加载（JSON）

### 3.2 预留模块（Phase 2+）

- [ ] 数据源连接（API/数据库）
- [ ] 自定义组件注册
- [ ] 多页面路由管理
- [ ] UI 主题/设计系统
- [ ] 协作与版本管理
