## 1. 范围与优先级说明

| 界面 | 本期状态 | 说明 |
|---|---|---|
| CLI 终端交互界面 | **P0，本期完整设计与实现** | 产品核心体验载体 |

## 2. CLI 终端交互界面设计（本期核心）

### 2.1 视觉规范

| 元素 | 规范 |
|---|---|
| 主色 | 青蓝色系 `#00B8D9`，用于Logo、成功状态 |
| 成功/警告/错误/信息 | 绿`✓` / 黄`⚠` / 红`✗` / 灰`ℹ`，无颜色终端降级为 `[OK]/[WARN]/[FAIL]/[INFO]` |
| 强调文本 | 加粗，避免大范围彩色刷屏 |

### 2.2 关键界面线框（本期实际实现版本，无授权拦截）

**① 启动欢迎（本期：无强制激活拦截）**
```
┌────────────────────────────────────────────┐
│  Jaicode v0.1.0 (Preview)                   │
│  ────────────────────────────               │
│  ℹ 当前为内测版本，暂不限制使用 / Preview build │
│                                              │
│  输入 `jaicode --help` 查看可用命令。          │
└────────────────────────────────────────────┘
```

**② Code 模式 — 任务执行与 Diff 确认（本期核心体验）**
```
$ jaicode code "修复登录接口空指针异常"

⠋ 正在读取项目上下文...
⠙ 正在分析 auth/login.ts...
✓ 已生成修复方案

──────────────────────────────────────────────
📝 变更文件: src/auth/login.ts
──────────────────────────────────────────────
  12 | function login(user) {
- 13 |   return user.token.value;
+ 13 |   return user?.token?.value ?? null;
  14 | }
──────────────────────────────────────────────

是否应用此变更？ / Apply this change? [y/N]: _
```

**③ Debug 模式 — 含重试上限的进度展示**
```
$ jaicode debug "npm test"

```

```

  可运行 `jaicode debug "npm test" --max-retries 8` 提高重试上限。

```

**⑤ 意向收集提示**
```

💡 Jaicode 目前处于内测阶段，未来将推出正式授权版本。
   如果这个工具对你有帮助，欢迎登记意向： https://jaicode.dev/waitlist

```

### 2.3 组件规范

| 组件 | 行为规范 |
|---|---|
| Spinner | 异步等待LLM响应时展示，超10秒提示"仍在处理" |
| 确认交互 `[y/N]` | 默认拒绝(N)，支持 `--yes` 跳过（CI场景） |
| Diff展示 | 统一Diff格式，超50行自动折叠 |
| 进度提示（Debug） | 显示"当前/最大"尝试次数 |
| 双语切换反馈 | 切换后立即用新语言输出确认信息 |

### 2.4 信息分级策略
- L1关键结果：始终显示
- L2过程信息：`--quiet` 可抑制
- L3调试信息：仅 `--verbose` 输出

## 3. Web 端占位设计（P1，极简版）

本期**不设计**完整的登录/授权管理/账单页面。仅需一个单页占位站点，用于承接CLI内的意向收集入口。

### 3.1 唯一页面：Waitlist 占位页

```
┌─────────────────────────────────────────────────────────┐
│                       Jaicode                            │
│         本地优先的中英双语终端 AI 编程助手（内测中）             │
│                                                             │
│              [ 输入邮箱登记意向______________ ]  [提交]        │
│                                                             │
│         代码不出本地设备 · 中英文体验对等 · 授权可离线激活         │

```

**功能范围**：仅邮箱收集+简单说明文案，无需登录态、无需数据库以外的复杂后端。

### 3.2 二期预告（本期不设计，仅记录待办）

以下页面**明确排除在本期设计范围外**，待授权/支付模块启动开发时重新细化：

- 控制台首页（授权状态/用量概览）
- 授权码管理（列表/设备绑定/离线证书生成）
- 团队与席位管理
- 账单与订阅

## 4. 国际化考量
- CLI与（未来）Web共用同一套语义化key的多语言资源
- 日期、数字格式遵循区域习惯

## 5. 验收标准（本期）

| 项目 | 标准 |
|---|---|
| CLI无颜色环境降级 | 不支持ANSI的终端下所有状态符号有纯文本兜底 |
| Diff确认默认值 | 所有写文件操作默认选项为"否" |
| Debug重试上限提示 | 达到上限时给出明确的人工介入提示及参数调整方法 |
| 回滚功能可用 | `jaicode rollback` 可正确恢复最近一次备份 |
| 意向收集非侵入 | 提示展示频率可配置，不干扰正常任务输出 |
| Waitlist页面 | 邮箱提交成功率、基础可用性即可，无需视觉高保真度 |
