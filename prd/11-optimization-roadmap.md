# Jaicode 优化方案 V1

> 基于能力符合度分析，制定分阶段优化路线图

---

## 第一阶段：安全合规加固（P0 — 最高优先级）

### 1.1 内容安全过滤层

**目标：** 拦截恶意内容，拦截率 >99%

**方案：**
- 在 LLM 调用前增加输入过滤（关键词 + 语义分类器）
- 在 LLM 返回后增加输出过滤（敏感信息检测）
- 使用开源安全模型（如 LlamaGuard）或规则引擎

**实现：**
```
packages/tui-node/src/security/
  ├── input-filter.js     — 输入清洗 + 意图分类
  ├── output-filter.js    — 输出敏感信息检测
  └── blocked-patterns.js — 黑名单规则
```

### 1.2 Prompt 注入防护

**目标：** 防止系统指令被用户输入覆盖

**方案：**
- System Prompt 与用户输入严格分离（使用消息角色边界）
- 检测并拒绝包含指令覆盖模式的输入（如 "ignore previous"、"system:"）
- 对用户输入中的特殊字符进行转义

### 1.3 隐私保护增强

**目标：** 防止敏感信息泄露

**方案：**
- 会话数据加密存储（AES-256）
- 自动检测并脱敏：身份证号、手机号、API Key、密码
- 会话过期自动清理（默认 1 小时）

## 第二阶段：核心能力增强（P1 — 高优先级）

### 2.1 智能意图识别升级

**目标：** 从关键词匹配升级为语义理解

**方案：**
- 使用轻量级本地模型（如 ONNX 格式的蒸馏模型）做意图分类
- 或调用 LLM 做二级分类（先理解意图，再路由到对应模式）
- 支持模糊表达、方言、中英混杂

### 2.2 多轮对话记忆

**目标：** 跨会话上下文保持

**方案：**
- 会话摘要生成（每 N 轮压缩一次上下文）
- 项目级记忆（`.jaicode/memory.yaml` 存储关键决策）
- 用户偏好学习（自动记录常用命令、偏好 Provider）

### 2.3 知识时效性

**目标：** 识别未知知识并引导用户

**方案：**
- 联网搜索集成（MCP web-search）
- 知识截止日期提示
- "未知" 检测 + 引导补充知识

## 第三阶段：生产适配（P2 — 中优先级）

### 3.1 API 化改造

**目标：** 支持多端接入和并发

**方案：**
```
packages/api/
  ├── server.js       — HTTP/WebSocket 服务
  ├── auth.js         — API Key 鉴权
  ├── rate-limiter.js — 限流熔断
  └── routes/
      ├── chat.js     — 对话接口
      ├── skills.js   — 技能管理
      └── health.js   — 健康检查
```

### 3.2 可观测性

**目标：** 生产级监控和排查能力

**方案：**
- 结构化日志（JSON 格式，支持 ELK 接入）
- 指标暴露（Prometheus /metrics 端点）
- 分布式追踪（OpenTelemetry 兼容）

### 3.3 多模态支持

**目标：** 图片理解能力

**方案：**
- 接入 VL 模型 MCP（如 Claude Vision）
- 图片路径检测 → 自动调用 VL 模型
- 返回结构化描述

---

## 第四阶段：用户体验优化（P3 — 持续迭代）

### 4.1 交互体验

- 输入历史（↑↓ 键浏览）
- 自动补全（命令、路径、Provider）
- 进度条（长时间操作可视化）

### 4.2 错误处理

- 分级错误提示（用户级 / 系统级 / 调试级）
- 自动重试（网络抖动场景）
- 优雅降级（Provider 不可用时切换备选）

### 4.3 多端一致性

- Web 终端（ttyd 集成）
- IDE 插件（VS Code / JetBrains）
- API SDK（Python / JS 客户端）
