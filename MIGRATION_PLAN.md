### 迁移阶段

```
Phase 1 (v0.14.x) — 安全基础
  ├── 输入过滤 (Prompt 注入检测 + 敏感信息识别)
  ├── 输出过滤 (API Key/手机号/邮箱脱敏)
  └── 安全沙箱 (路径遍历 + 执行限制)

Phase 2 (v0.15.x) — 智能上下文
  ├── 上下文压缩 (Token 估算 + LCS 摘要)
  ├── 项目记忆 (自动扫描 + memory.yaml)
  ├── 用户画像 (user.profile + 习惯学习)
  └── 知识时效检测 (KNOWLEDGE_CUTOFF + 时间关键词)

Phase 3 (v0.16.x) — 运营能力
  ├── 审计日志 (JSONL 持久化 + 查询)
  ├── 用量统计 (Token 计数 + 请求数 + 延迟 P95)
  ├── 自动更新检查 (GitHub Release API)
  └── 多 Provider 热切换 (配置热加载)

Phase 4 (v0.17.x) — API + 集成
  ├── HTTP API 服务器 (REST + SSE + gRPC)
  └── TypeScript 前端→Go 后端通信适配

Phase 5 (v0.18.x) — 切换准备
  ├── 集成测试
  ├── 性能基准
  └── 生产切换演练
```

## 三、生产切换策略

### 切换前检查清单

- [ ] 所有 TypeScript 功能 Go 已实现
- [ ] 单元测试覆盖率 >80%
- [ ] 集成测试全部通过
- [ ] 性能基准不低于 TypeScript 版本
- [ ] M1/M2/M3/M4/M5 全部兼容
- [ ] 压力测试通过（100 并发请求）
- [ ] 内存泄漏检测通过（24h 运行）
- [ ] 数据迁移脚本准备完毕

### 切换步骤

```
1. Go 后端二进制部署到生产服务器
2. 启动 Go 后端 (localhost:3003)
3. 验证 Go 后端健康状态
4. TypeScript TUI 切换连接到 Go 后端
5. 监控 24 小时
6. 确认无问题后停止 TypeScript 后端
```

### 回滚方案

```
问题出现时:
1. 停止 Go 后端
2. TypeScript TUI 切换回本地模式
3. TypeScript 后端重新启动
4. 分析问题并修复
5. 修复后重新尝试切换
```
