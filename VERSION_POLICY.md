# Jaicode 双语言版本策略

## 分支版本对应关系

| 分支 | 后端语言 | 当前版本 | 目标用户 |
|---|---|---|---|
| `main` | TypeScript | v0.13.0 | 生产用户 |
| `feat/go-backend` | Go | v0.14.0 | 开发/早期采用者 |

## 版本号规则（宪法约束）

### 通用规则
- 每次只递增一个位置
- 补丁：0.13.0 → 0.13.1（bug 修复）
- 次要：0.13.0 → 0.14.0（新功能）
- 主要：0.x → 1.0.0（架构变更）
- **禁止跳号**

### TypeScript 分支（main）
- 从 v0.13.0 开始
- 每次递增一个位置

### Go 分支（feat/go-backend）
- 从 v0.14.0 开始（与 TypeScript 并行）
- 每次递增一个位置
- 功能对齐 TypeScript 后合并回 main

## 同步策略

1. TypeScript 分支的 bug 修复同步到 Go 分支
2. Go 分支的新功能在验证后合并回 main
3. 两个分支的 TUI 前端保持同步
4. 版本号独立递增，合并时取较高者
