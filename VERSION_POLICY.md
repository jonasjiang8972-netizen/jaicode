# Jaicode 版本策略

> 项目保持持续优化状态。版本号反映功能演进。

## 版本号规则

```
v{major}.{minor}.{patch}

- patch (第三位): bug 修复、UI 优化
  → v0.15.1 → v0.15.2

- minor (第二位): 新功能、新能力
  → v0.15.0 → v0.16.0

- major (第一位): 架构变更
  → v0.x.x → v1.0.0

⚠️ 禁止跳号：每次只递增一个位置
```
