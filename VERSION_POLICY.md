# Jaicode 双语言版本策略

> 项目保持永久优化状态，不推出正式版。版本号反映功能演进。

---

## 版本号规则

```
v{major}.{minor}.{patch}

- patch (第三位): bug 修复、UI 优化、小调整
  → v1.0.0 → v1.0.1

- minor (第二位): 新功能、新能力、新形态
  → v1.0.0 → v1.1.0

- major (第一位): 架构变更、破坏性更新
  → v0.x.x → v1.0.0

⚠️ 禁止跳号：每次只递增一个位置
```

## 版本对齐策略

- 功能对齐：TypeScript 与 Go 功能 1:1 同步
- 独立版本：各分支独立递增
- 合并取高：功能合并时取较高版本号
- 用户透明：版本差异对用户无感知
