# ADR-001: 核心能力层 - 文件读写与命令执行

## 状态
Accepted

## 背景
Jaicode v0.4.0 仅有 LLM 对话能力，AI 无法直接读取项目文件、执行命令。这是 P0 缺失能力，导致 AI 只能给出建议而无法实际操作。

## 决策
采用 **Skill 模块化架构** 而非 MCP 协议（MVP 阶段）：
- 文件读写和命令执行作为内置 Skill，不依赖外部 MCP Server
- 安全机制：路径遍历防护 + 危险命令黑名单 + 超时限制
- 通过 `/read` 和 `/exec` 命令在 TUI 中直接调用

## 后果
- **正面**：零依赖、启动即用量、终端友好
- **负面**：功能比完整 MCP 协议有限，后续可通过 MCP 网关扩展

## 实现
1. `skills/file-reader.js` - 安全读取、路径校验、上下文注入
2. `skills/shell-executor.js` - 执行命令、安全检查、输出捕获
3. `tui.js` 集成 - /read /exec 命令 + 自动项目扫描
